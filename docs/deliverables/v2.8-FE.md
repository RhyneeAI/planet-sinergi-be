## v2.7 (Sudah Rilis — Role Restructure, OPS Transfer, ABS)

### Data Model — Transfer Flow

```
┌────────────────────────────────────────────────────────────┐
│                      ADMIN (PUSAT)                         │
├────────────────────────────────────────────────────────────┤
│  OpsExpense (expense_type = MANDOR)                        │
│    ├── mandor_id: (mandor target)                          │
│    ├── transferConfirmation → status: PENDING/CONFIRMED    │
│    └── transferIncome → OpsIncome (created saat confirm)   │
└────────────────────────────────────────────────────────────┘
                           │
                     (mandor confirm)
                           ▼
┌────────────────────────────────────────────────────────────┐
│                      MANDOR                                │
├────────────────────────────────────────────────────────────┤
│  OpsIncome (source_type = MANDOR)                          │
│    ├── mandor_id: (mandor ID)                              │
│    └── muncul di GET /api/v1/incomes (filter by mandor)    │
└────────────────────────────────────────────────────────────┘
```

**Alur:**
1. Admin buat transfer → OpsExpense (MANDOR) + OpsTransferConfirmation (PENDING)
2. Mandor confirm via `POST .../transfer-confirmations/{uuid}/confirm` → OpsIncome terbuat, status jadi CONFIRMED
3. Mandor reject → OpsExpense di-soft-delete, income tidak pernah dibuat

## v2.8

### 1. Login — MARKETING & MARKETING_LEAD Diblokir

```json
}
```

### 2. New Standalone Endpoint — `/api/v1/marketings`

Akses data marketing lintas modul (tanpa prefix `/operational` atau `/pos`).

| Method | Endpoint | Auth | Deskripsi |
|--------|----------|------|-----------|
| GET | `/api/v1/marketings` | SUPERADMIN,OWNER,ADMIN,KEPALA_GUDANG,KEPALA_MANDOR,GUDANG | List marketing |
| GET | `/api/v1/marketings/{uuid}` | sama | Detail marketing + hierarchy |
| POST | `/api/v1/marketings` | SUPERADMIN,ADMIN,KEPALA_GUDANG,KEPALA_MANDOR | Create marketing |
| PUT | `/api/v1/marketings/{uuid}` | sama | Update marketing |
| DELETE | `/api/v1/marketings/{uuid}` | sama | Delete marketing |

**Query params (index):**
- `?role=MARKETING_LEAD` — filter by role
- `?leader_uuid={uuid}` — ambil anggota dari LEAD tertentu
- `?search=` — cari by name/phone/email

**Response (index):**
```json
{
  "data": [
    {
      "role": "MARKETING_LEAD",
      "leader": null,
      "members_count": 5
    },
    {
      "role": "MARKETING",
      "leader": { "uuid": "...", "name": "Leader Name" },
      "members": null,
      "members_count": 0
    }
  ]
}
```

**Response (show — MARKETING_LEAD):**
```json
{
  "data": {
    "role": "MARKETING_LEAD",
    "members": [
      { "uuid": "...", "name": "Anggota 1" }
    ],
    "members_count": 2
  }
}
```

**Notes:**
- `leader` hanya muncul jika role = MARKETING
- `members` + `members_count` hanya muncul jika role = MARKETING_LEAD
- Saat create MARKETING, kirim `leader_uuid` untuk assign ke LEAD
