## 1. New Standalone Endpoints

### /api/v1/marketings (Cross-Module)

Akses data marketing tanpa prefix `/operational` atau `/pos`.

| Method | Endpoint | Auth | Deskripsi |
|--------|----------|------|-----------|
| GET | `/api/v1/marketings` | SUPERADMIN,OWNER,ADMIN,KEPALA_GUDANG,KEPALA_MANDOR,GUDANG | List marketing |
| GET | `/api/v1/marketings/{uuid}` | sama | Detail marketing + hierarchy |
| POST | `/api/v1/marketings` | SUPERADMIN,ADMIN,KEPALA_GUDANG,KEPALA_MANDOR | Create marketing |
| PUT | `/api/v1/marketings/{uuid}` | sama | Update marketing |
| DELETE | `/api/v1/marketings/{uuid}` | sama | Delete marketing |

**Query params (index):**
- `?role=MARKETING_LEAD` — filter by role
- `?leader_uuid={uuid}` — ambil anggota dari LEAD tertentu
- `?search=` — cari by name/phone/email

**Response (index):**
```json
{
  "data": [
    {
      "uuid": "...",
      "name": "...",
      "phone": "...",
      "role": "MARKETING_LEAD",
      "leader": null,
      "members_count": 5,
      "is_active": true
    },
    {
      "uuid": "...",
      "name": "...",
      "role": "MARKETING",
      "leader": { "uuid": "...", "name": "Leader Name" },
      "members": null,
      "members_count": 0,
      "is_active": true
    }
  ]
}
```

**Response (show — MARKETING_LEAD):**
```json
{
  "data": {
    "uuid": "...",
    "name": "...",
    "role": "MARKETING_LEAD",
    "members": [
      { "uuid": "...", "name": "Anggota 1" },
      { "uuid": "...", "name": "Anggota 2" }
    ],
    "members_count": 2
  }
}
```

**Notes:**
- `leader` hanya ada jika role = MARKETING
- `members` + `members_count` hanya ada jika role = MARKETING_LEAD
- Saat create MARKETING, kirim `leader_uuid` untuk assign ke LEAD
- Saat update MARKETING, kirim `leader_uuid` untuk ganti LEAD

## 2. Expense Detail — `transfer_confirmation`

Endpoint `GET /api/v1/operational/expenses/{uuid}` (dan index) sekarang return `transfer_confirmation` untuk MANDOR-type expenses:

```json
{
  "data": {
    "uuid": "...",
    "expense_type": "MANDOR",
    "amount": 500000,
    "transfer_confirmation": {
      "uuid": "...",
      "status": "CONFIRMED",
      "confirmed_amount": 500000,
      "confirmed_at": "2026-06-30T10:00:00.000000Z",
      "confirmed_by": { "uuid": "...", "name": "Mandor A" }
    },
    "transfer_income": {
      "uuid": "...",
      "amount": 500000,
      "source_type": "MANDOR"
    }
  }
}
```

**Status values:** `"PENDING"` | `"CONFIRMED"` | `"REJECTED"`

**Notes:**
- `transfer_confirmation` = status transfer (ada di expense langsung)
- `transfer_income` = income yang terbuat saat confirm (belum ada jika PENDING)
- Sebelum fix ini, `transfer_confirmation` tidak muncul di response → sekarang muncul

## 3. Dashboard Mandor — Remaining Amount

`GET /api/v1/operational/dashboard/mandor`

**Perubahan:** `total_expense` sekarang **tidak termasuk** MANDOR-type expenses (transfer dari pusat). Sehingga `remaining_amount` menjadi lebih besar (correct).

Sebelum: `remaining_amount = total_income - total_expense (termasuk transfer masuk)`  
Sesudah: `remaining_amount = total_income - total_expense (hanya expense INTERNAL)`

## 5. Data Model — Transfer Flow

```
┌────────────────────────────────────────────────────────────┐
│                      ADMIN (PUSAT)                         │
├────────────────────────────────────────────────────────────┤
│  OpsExpense (expense_type = MANDOR)                        │
│    ├── mandor_id: (mandor target)                          │
│    ├── transferConfirmation → status: PENDING/CONFIRMED    │
│    └── transferIncome → OpsIncome (created saat confirm)   │
└────────────────────────────────────────────────────────────┘
                           │
                     (mandor confirm)
                           ▼
┌────────────────────────────────────────────────────────────┐
│                      MANDOR                                │
├────────────────────────────────────────────────────────────┤
│  OpsIncome (source_type = MANDOR)                          │
│    ├── mandor_id: (mandor ID)                              │
│    └── muncul di GET /api/v1/incomes (filter by mandor)    │
└────────────────────────────────────────────────────────────┘
```

**Alur:**
1. Admin buat transfer → OpsExpense (MANDOR) + OpsTransferConfirmation (PENDING)
2. Mandor confirm via `POST .../transfer-confirmations/{uuid}/confirm` → OpsIncome terbuat, status jadi CONFIRMED
3. Mandor reject → OpsExpense di-soft-delete, income tidak pernah dibuat

Ada 3 cara akses data marketing — pilih sesuai kebutuhan:

| Prefix | Endpoint | Hierarchy? | Cocok untuk |
