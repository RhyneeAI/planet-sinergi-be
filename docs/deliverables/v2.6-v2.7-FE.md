# Gudang Planet — FE Deliverable v2.6 & v2.7

## v2.6 — KEPALA_MANDOR

### 1. Role Baru: KEPALA_MANDOR

| Role | Posisi |
|------|--------|
| `KEPALA_MANDOR` | Mandor dengan akses penuh — di atas `MANDOR`, setara `KEPALA_GUDANG` di POS |

**Behavior:**
- Di POS: akses penuh (read + write semua master data, transaksi, laporan)
- Di Operasional: akses seperti MANDOR (pemasukan/pengeluaran cabang, dompet, transfer confirmation/reject reports income-expense)
- Auth response: sama seperti role lain (token + user data)

### 2. Reports with Proof Files

Endpoint baru di `/api/v1/reports/` — khusus Operasional:

| Endpoint | Method | Auth | Deskripsi |
|----------|--------|------|-----------|
| `/api/v1/reports/income-expense` | GET | `SUPERADMIN,OWNER,ADMIN,KEPALA_MANDOR` | Summary income/expense per periode |
| `/api/v1/reports/income-expense/detail` | GET | `SUPERADMIN,OWNER,ADMIN,KEPALA_MANDOR` | Detail transaksi per cabang |
| `/api/v1/reports/income-expense/download` | GET | `SUPERADMIN,OWNER,ADMIN,KEPALA_MANDOR` | Download PDF summary |

**Query params (sama untuk 3 endpoint):**
- `date_from` (Y-m-d, optional, default: awal bulan)
- `date_to` (Y-m-d, optional, default: hari ini)
- `sub_company_uuid` (optional, filter per cabang)

**Response income-expense/detail:**
```json
{
  "success": true,
  "message": "...",
  "data": [
    {
      "uuid": "...",
      "name": "...",
      "amount": 50000,
      "date": "2026-06-25",
      "type": "income",
      "category": "INTERNAL",
      "payment_method": "CASH",
      "proof_files": ["https://.../proof1.jpg", "https://.../proof2.jpg"],
      "sub_company": { "uuid": "...", "name": "Cabang A" },
      "created_by": { "uuid": "...", "name": "Admin" }
    }
  ]
}
```

**Download:** `/api/v1/reports/income-expense/download` — response langsung download PDF (Content-Type: application/pdf).

## v2.7 — POS Role Restructure

### 1. Perubahan Role

| Sebelum (v2.6) | Sesudah (v2.7) | Keterangan |
|----------------|----------------|------------|
| `MANAGER_GUDANG` | `GUDANG` | Namanya berubah. Hanya akses produk, kategori, satuan, stok, laporan |
| — | `KEPALA_GUDANG` | **Baru**. Akses penuh POS (setara KEPALA_MANDOR) |
| `KEPALA_MANDOR` | `KEPALA_MANDOR` | Tidak berubah — tetap akses penuh POS |

| OWNER | R | R | R | R | R | R | R | — | R | R |
| ADMIN | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | — | ✅ |
| KEPALA_GUDANG | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |

### 3. Marketing — Pindah ke OPS (Dual Gateway)

#### POS (Read-only)
- `GET /api/v1/marketings`
- `GET /api/v1/marketings/{uuid}`
- Store/update/destroy **dihapus** dari POS

#### OPS (Full CRUD) — Baru
Base: `/api/v1/operational/marketings`

| Method | Endpoint | Auth | Deskripsi |
|--------|----------|------|-----------|
| GET | `/api/v1/operational/marketings` | `SUPERADMIN,OWNER,ADMIN,KEPALA_GUDANG,KEPALA_MANDOR,GUDANG` | List marketing |

### 4. Perubahan Routing POS

Ringkasan perubahan role middleware:

| Group | v2.6 | v2.7 |
|-------|------|------|
| Products generate-code | `SUPERADMIN,ADMIN,MANAGER_GUDANG` | `SUPERADMIN,ADMIN,GUDANG,KEPALA_GUDANG,KEPALA_MANDOR` |
| Master data READ | `SUPERADMIN,OWNER,ADMIN,MANAGER_GUDANG` | `SUPERADMIN,OWNER,ADMIN,KEPALA_GUDANG,KEPALA_MANDOR,GUDANG` |
| Stock write | same as master write | `SUPERADMIN,ADMIN,KEPALA_GUDANG,KEPALA_MANDOR,GUDANG`* |
| Non-stock write | `SUPERADMIN,ADMIN,MANAGER_GUDANG` | `SUPERADMIN,ADMIN,KEPALA_GUDANG,KEPALA_MANDOR`** |
| Transactions | `SUPERADMIN,OWNER,ADMIN,MANAGER_GUDANG,KASIR` | `SUPERADMIN,OWNER,ADMIN,KEPALA_GUDANG,KEPALA_MANDOR,KASIR` |
| Purchase | `SUPERADMIN,ADMIN,MANAGER_GUDANG` | `SUPERADMIN,ADMIN,KEPALA_GUDANG,KEPALA_MANDOR` |
| Reports | `SUPERADMIN,OWNER,ADMIN,MANAGER_GUDANG,...` | `SUPERADMIN,OWNER,ADMIN,GUDANG,KEPALA_GUDANG,KEPALA_MANDOR,...` |

*\* Stock write: hanya produk, kategori, satuan, stock-mutation.*
*\*\* Non-stock write: supplier, customer, customer-type, marketing-product.*
