### Catatan Behavior per Role (Controller Logic)

| Aspek | ADMIN / SUPERADMIN / OWNER | MANDOR / KEPALA_MANDOR |
|-------|---------------------------|----------------------|
| **Income store** | `source_type: INTERNAL` — pemasukan pusat | `source_type: INTERNAL` — pemasukan cabang, wajib `sub_company_uuid` |
| **Expense store** | `expense_type: INTERNAL` (pusat) atau `expense_type: MANDOR` (transfer → buat income pending + transfer confirmation) | `expense_type: INTERNAL` — pengeluaran cabang, auto-debit wallet |
| **Scope data** | Lihat semua data dalam company | Hanya lihat data milik sendiri / mandor bawahan (KEPALA_MANDOR) |
| **Transfer confirmation** | Hanya read | Confirm / Reject + adjust `confirmed_amount` |
| **Wallet** | Tidak akses | Lihat saldo + riwayat transaksi |

## Aplikasi POS

**Role yang login:** `SUPERADMIN`, `OWNER`, `ADMIN`, `KEPALA_MANDOR`, `KEPALA_GUDANG`, `GUDANG`, `KASIR`

| Group | Role | Akses |
|-------|------|-------|
| Product generate-code | `SUPERADMIN`, `ADMIN`, `GUDANG`, `KEPALA_GUDANG`, `KEPALA_MANDOR` | Generate code |

| Sales transactions + installments + returns | `SUPERADMIN`, `OWNER`, `ADMIN`, `KEPALA_GUDANG`, `KEPALA_MANDOR`, `GUDANG`, `KASIR` | CRUD |
| Purchase transactions + installments | `SUPERADMIN`, `ADMIN`, `KEPALA_GUDANG`, `KEPALA_MANDOR` | CRUD |
