# v1.9 ÔÇö Absensi: HRD, Lembur, Kasbon, & Employee Detail

## Role Baru

| Role | Enum | Deskripsi |
|---|---|---|
| HRD | `HRD` | Admin khusus Absensi; melihat semua karyawan + gaji; mengelola lembur & kasbon |

## Rule Akses

- **MARKETING** dan **MARKETING_TETAP** tidak perlu absensi & payroll (bukan karyawan tetap)
- **HRD** bisa melihat payroll history (read-only), tidak bisa create/update

## Overtime (Lembur)

### Table: `abs_overtimes`

| Field | Type | Notes |open
|---|---|---|
| id | bigint PK | |
| user_id | bigint FK Ôćĺ users | |
| date | date | Tanggal lembur |
| start_time | time | Jam mulai |
| end_time | time | Jam selesai |
| reason | text | Alasan |
| status | enum | `pending`, `approved`, `rejected` |
| approved_by | bigint FK Ôćĺ users, nullable | HRD yg ACC |
| company_id | bigint FK Ôćĺ companies | |

### Alur
1. Karyawan / HRD input lembur (batch, pilih beberapa karyawan) Ôćĺ status `pending`
2. HRD ACC Ôćĺ `approved`, isi `approved_by`
3. Data yang `approved` dipakai kalkulasi payroll (rate dari `custom_configurations.overtime_hourly_rate`)

### API

| Method | Endpoint | Auth | Notes |
|---|---|---|---|
| GET | `/api/v1/abs/overtimes` | HRD, Admin | List, filter status/date/user |
| POST | `/api/v1/abs/overtimes` | HRD | Batch input (array `user_ids`) |
| GET | `/api/v1/abs/overtimes/{id}` | HRD, Admin | Detail |
| PUT | `/api/v1/abs/overtimes/{id}/approve` | HRD | ACC |
| PUT | `/api/v1/abs/overtimes/{id}/reject` | HRD | Tolak (body: `reason`) |

## Kasbon (Loan)

### Table: `abs_loans`

| Field | Type | Notes |
|---|---|---|
| id | bigint PK | |
| user_id | bigint FK Ôćĺ users | |
| amount | decimal | Total pinjaman |
| reason | text | Kebutuhan |
| tenor_months | int | 1ÔÇô4 |
| monthly_installment | decimal | `amount / tenor` |
| remaining_balance | decimal | Sisa cicilan |
| status | enum | `pending`, `approved`, `rejected`, `paid` |
| approved_by | bigint FK Ôćĺ users, nullable | |
| company_id | bigint FK Ôćĺ companies | |

### Alur
1. HRD input kasbon atas nama karyawan Ôćĺ `pending`
2. HRD ACC Ôćĺ `approved`, `remaining_balance = amount`
3. Saat payroll: potong `monthly_installment` dari gaji, kurangi `remaining_balance`
4. Jika `remaining_balance <= 0` Ôćĺ `paid`

### API

| Method | Endpoint | Auth | Notes |
|---|---|---|---|
| GET | `/api/v1/abs/loans` | HRD, Admin | List |
| POST | `/api/v1/abs/loans` | HRD | Create |
| GET | `/api/v1/abs/loans/{id}` | HRD, Admin | Detail |
| PUT | `/api/v1/abs/loans/{id}/approve` | HRD | ACC |
| PUT | `/api/v1/abs/loans/{id}/reject` | HRD | Tolak |
